# Section 04: Markdown Export (`server/export.py` and `tests/test_export.py`)

## Overview

This section implements markdown document generation from a `ResearchResult` object. The `generate_markdown()` function produces a complete, well-structured Markdown document containing all research artifacts and source evaluations. A `topic_slug()` helper produces safe filenames from arbitrary topic strings (including Unicode). These are consumed by the export route (section-06) to serve downloadable `.md` files.

## Pipeline Models Reference

The export function works with models from `beacon.models` (in `01-agent-pipeline/beacon/models.py`). The key types are:

```python
class EvaluatedSource(BaseModel):
    url: str
    title: str
    snippet: str
    signals: IntelligenceSignals
    deep_read_content: str | None = None
    extraction_method: str | None = None

class IntelligenceSignals(BaseModel):
    learning_efficiency_score: int  # 0-10
    content_type: Literal["tutorial", "paper", "docs", "opinion", "video",
                          "forum", "repository", "course", "other"]
    time_estimate_minutes: int
    recency: str | None
    key_insight: str
    coverage: list[str]
    evaluation_failed: bool = False

class Flashcard(BaseModel):
    question: str
    answer: str

class ResearchResult(BaseModel):
    topic: str
    depth: str
    sources: list[EvaluatedSource]
    artifacts: dict[str, Any]
    session_id: str
    timestamp: str
```

The `artifacts` dict typically contains keys `"summary"`, `"concept_map"`, `"flashcards"`, and `"resources"`, but values can be strings, lists of Flashcard objects, JSON strings, or None. The export function must handle all these type variations defensively.

## Implementation Details

### File: `C:\git_repos\playground\hackathon\02-api-streaming\server\export.py`

#### `topic_slug(topic: str) -> str`

Converts an arbitrary topic string into a safe filename slug.

Algorithm:
1. Apply `unicodedata.normalize('NFKD', topic)` to decompose Unicode characters
2. Encode to ASCII with `errors='ignore'` and decode back to get ASCII-only string
3. Lowercase the result
4. Replace any run of characters matching `[^a-z0-9]+` with a single hyphen using `re.sub`
5. Strip leading and trailing hyphens
6. Truncate to 50 characters
7. Strip any trailing hyphens that truncation might have exposed

Required imports: `unicodedata`, `re`.

#### `generate_markdown(result: ResearchResult) -> str`

Builds a complete Markdown document as a string. Takes a `ResearchResult` and returns the Markdown string.

Required imports: `json` (for parsing JSON-string artifacts), `datetime` (for formatting date), and `beacon.models` types (`ResearchResult`, `EvaluatedSource`, `Flashcard`).

**Document structure** (built by appending sections to a list of strings, then joining with newlines):

1. **Header section**:
   - `# Research: {result.topic}`
   - A metadata line: `**Depth:** {result.depth} | **Sources:** {len(result.sources)} | **Date:** {formatted date from result.timestamp}`

2. **Executive Summary section** (`## Executive Summary`):
   - Get `result.artifacts.get("summary")`. If the value is None or the key is missing, render `*(Not generated)*`. Otherwise render the value as-is (call `str()` if not already a string).

3. **Sources section** (`## Sources (Ranked by Learning Efficiency)`):
   - If `result.sources` is empty, render a note like `*No sources evaluated.*`
   - Otherwise, sort sources by `source.signals.learning_efficiency_score` descending
   - For each source, render a subsection (`### {source.title}`) containing:
     - URL as a markdown link
     - Score badge: `**Score:** {score}/10`
     - Content type: `**Type:** {signals.content_type}`
     - Time estimate: `**Estimated read time:** {signals.time_estimate_minutes} min`
     - Recency: `**Recency:** {signals.recency}` (only if not None)
     - Key insight: `**Key insight:** {signals.key_insight}`
     - Coverage: `**Topics covered:** {', '.join(signals.coverage)}`

4. **Concept Map section** (`## Concept Map`):
   - Get `result.artifacts.get("concept_map")`. Handle None/missing with `*(Not generated)*`. Otherwise render as-is.

5. **Flashcards section** (`## Flashcards`):
   - Get `result.artifacts.get("flashcards")`. Handle None/missing with `*(Not generated)*`.
   - If it is a `list` and items have a `.question` attribute (i.e., `Flashcard` objects), iterate and format as numbered Q&A pairs: `**Q{n}:** {question}` / `**A{n}:** {answer}`.
   - If it is a `str`, attempt `json.loads()` to parse into a list of dicts. On success, iterate and format the same way using dict key access (`item["question"]`, `item["answer"]`). On failure (`json.JSONDecodeError`), render the raw string.
   - If it is a `list` of dicts (not Flashcard objects), iterate and format using dict key access.

6. **Resources section** (`## Resources`):
   - Get `result.artifacts.get("resources")`. Handle None/missing with `*(Not generated)*`.
   - If it is a `str`, attempt `json.loads()`. On success, render the structured data (format depends on what the pipeline produces -- could be a list, dict, etc.; render readably). On failure, render the raw string.
   - If already a list or dict, render readably.

7. **Footer**:
   - A horizontal rule (`---`) followed by `*Generated by Beacon Research Agent*`

**Key principles**:
- Only render sections for the known artifact keys: `summary`, `concept_map`, `flashcards`, `resources`. Ignore any unknown keys.
- Every artifact access should use `.get()` with a None fallback and check for None before rendering.
- Never crash on unexpected types -- use `str()` as a last resort.
- Sources are always sorted by `learning_efficiency_score` descending before rendering.
